# Session Report — 2026-03-12 (Session 11)

## Session Focus
Embedding Model Evaluation: `gemini-embedding-001` vs `voyage-4-large`

## Duration
~4 hours (14:00 – 18:39 UTC+4)

## Objective
Determine whether to migrate embedding model from Gemini to Voyage for Georgian tax code vector search. Voyage API was already integrated for reranking — evaluate if Voyage embeddings offer better accuracy or latency.

## Workflow Executed

### Move 1: Deep Analysis ✅
- Researched Voyage model portfolio for Georgian language support
- Identified `voyage-4-large` as best candidate (1024-dim, multilingual)
- Hypothesized dimension mismatch risks (3072 vs 1024)

### Move 2: Strategic Planning ✅
- Designed 2-phase benchmark architecture:
  - Phase 1: Quick validation (10 queries, 48 articles)
  - Phase 2: Statistical significance (50 queries, 132 articles)
- Blueprint: hybrid 3-source golden dataset (real users, LLM-generated, edge cases)
- Defined metrics: P@1, P@5, MRR (strict + relaxed), latency percentiles

### Move 3: Execution ✅
- **Phase 1**: Built targeted corpus (8 golden + 40 hard negatives), ran benchmark
  - Gemini: MRR=0.82, P@1=7/10, latency=893ms
  - Voyage: MRR=0.75, P@1=6/10, latency=302ms
- **Phase 2**: Built 50-query golden set from MongoDB real queries, LLM generation, and manual edge cases
  - Curated 20 real user queries from 598 conversations in MongoDB
  - Generated 20 per-article queries covering all major tax code chapters
  - Crafted 10 edge cases (transliteration, cross-chapter, tricks)
  - Corpus: 132 articles (32 golden + 100 hard negatives)
  - Gemini: MRR=0.4231, P@1=15/50, latency=362ms
  - Voyage: MRR=0.3705, P@1=14/50, latency=305ms

### Move 4: QA Review ✅
- MRR diff = +5.26% absolute (+14% relative) → Gemini wins
- Per-source analysis: Gemini dominates real user + LLM queries; Voyage wins edge cases only
- Latency gap: narrowed from 3x to 1.2x
- Both models fail on multi-article/administrative queries → reranker is critical

### Session Closeout ✅
- Archived benchmark scripts to `/tmp/scoop-benchmark-archive-20260312/`
- Created ADR-001: `adr-001-embedding-model-selection.md`
- Updated `activeContext.md`

## Decision
**KEEP `gemini-embedding-001`** — Superior accuracy on Georgian tax queries justifies the marginal latency cost. Voyage API retained exclusively for reranking.
